# ✦ Kumpas

A high-performance financial tracking and forecasting ecosystem designed for Philippine college students. The platform abstracts advanced quantitative risk analysis and time-series behavioral machine learning into an accessible, real-world dashboard that prevents early budget depletion.

---

## System Overview

Kumpas is an integrated, microservices-driven platform built using a monorepo workspace. It bridges structured relational database records with a low-latency Python analytics sidecar. Rather than relying on simple static monthly spreadsheets, Kumpas dynamically projects financial runways by evaluating everyday historical spending behaviors against a student's unique allowance drop cycle. Translating the heavy complex finance math into user-friendly views.

---

## Problem Statement

Traditional expense trackers are entirely retrospective—they tell users where their money _went_, but fail to accurately project where their balance is _heading_. Students face structural spending surges (e.g., immediate outlays post-allowance drop, weekend social spikes) and uneven cycle lengths that distort simple averages. Without automated predictive foresight, students frequently experience rapid, unanticipated wallet depletion before their next scheduled allowance.

---

## Core Features

- **Allowance Cycle Tracking:** Establishes custom, flexible time boundaries based on real allowance drop frequencies rather than strict calendar months. It tracks active days elapsed and computes real-time horizons until the next incoming cash drop.
- **Granular Transaction Inputs:** Captures historical inflows and outflows tagged with precise metadata, including relational fields for category classifications ("FOOD", "COMMUTE", "DORM", "SCHOOL", "ADJUSTMENT", "ALLOWANCE") and specific merchant names.
- **Multi-Wallet Architecture:** Manages several active wallet instances per user, aggregating total real-time balances to serve as the unified liquidity baseline ($I_0$) for forward-looking simulations.
- **Simulator Shock Stress-Testing:** Introduces random, unexpected spending fluctuations into future timelines, allowing the core engine to evaluate how well a student's wallet can absorb spontaneous or emergency financial strains.
- **Cycle-Phase Behavioral Profiling (ML Pillar):** Automatically translates absolute calendar timestamps into normalized relative phase position indices (0.0 to 1.0) mapped inside a student’s specific allowance timeline.
- **Regularized Spend Regression:** Trains a Python Ridge Regression model using One-Hot Encoded spending categories to map localized spending surges, non-linear burn acceleration curves, and weekend spikes.
- **Vectorized Monte Carlo Runway Simulator (Quant Pillar):** Simulates 1,000 distinct forward-looking timeline trajectories via parallelized Arithmetic Brownian Motion. It pairs historical standard deviations ($\sigma$) with the dynamic drift predictions ($\mu$) generated by the machine learning model.
- **Fintech Weather Radar UI:** Translates multi-quantile probability distributions into intuitive, user-friendly, accessible indicators (e.g., Runway Success Rate %, Normal Daily Pace, Unpredictability Factor, and Financial Weather Gauges).
- **Bayanihan Collaborative Platform:** A dedicated micro-portal built around regional values of community cooperation, allowing group-based coordination and shared visibility for student goals. (To be developed soon)

## Tech Stack

### Frontend & Client Workspace

- **Framework:** React with Next.js
- **Routing Engine:** TanStack Router (File-based, Type-safe routing layout)
- **Styling Architecture:** Tailwind CSS
- **Iconography:** Lucide React

### Backend Systems

- **Core Application API:** NestJS Framework (TypeScript)
- **Database Management:** Prisma ORM with PostgreSQL
- **Authentication Subsystem:** Supabase Auth Guard Architecture
- **HTTP Communication Handler:** Axios Wrapper (`@nestjs/axios`)

### Machine Learning & Analytics Sidecar

- **Web Engine:** FastAPI (Python)
- **Server Server Gateway:** Uvicorn
- **Numerical Mathematics & Linear Algebra:** NumPy
- **Machine Learning Pipeline:** Scikit-Learn (One-Hot Encoder, Ridge Regression Models)

## Installation and Setup

### Prerequisites

- Node.js (v18 or higher)
- pnpm package manager (`npm install -g pnpm`)
- Python (v3.10 or higher) with `pip` and `virtualenv`

### Step 1: Clone and Configure Environment Files

Set up your `.env` flags within `apps/api` referencing your active PostgreSQL connection string and Supabase JWT credentials.

### Step 2: Install Node Workspace Dependencies

From the absolute root of the repository, execute:

```bash
pnpm install

```

### Step 3: Configure and Initialize the Python Virtual Environment

Navigate to the analytics directory to lock down Python dependencies:

```bash
cd apps/analytics
python -m venv .venv

# Activation (MacOS/Linux)
source .venv/bin/activate
# Activation (Windows)
.venv\Scripts\activate

pip install -r requirements.txt

```

### Step 4: Synchronize Database Models

Generate your Prisma schemas and push configurations directly to your local database engine:

```bash
pnpm --filter api prisma db push

```

### Step 5: Run the Monorepo Development Environment

To boot the unified frontend interface, NestJS server, and automated watch processes concurrently, run:

```bash
pnpm dev

```

In a parallel terminal window, ensure the Python mathematical sidecar is up and active:

```bash
cd apps/analytics
source .venv/bin/activate
python main.py

```

Open your browser and navigate to `http://localhost:5173` (or your terminal's designated Vite port) to interact with the system dashboard. Selecting **Radar Analytics** will automatically fire the historical extraction bridge, train your local profile model, and compute forward runway metrics instantly.
